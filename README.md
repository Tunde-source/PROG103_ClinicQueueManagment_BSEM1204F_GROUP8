-----

## 📌 Project Overview

The **Clinic Queue Management System** is a GUI-based Python application built with `tkinter`.  
It addresses the real-world problem of **disorganised patient queuing** in clinics across Sierra Leone,  
where long and chaotic queues reduce the quality and speed of healthcare delivery.

This system digitalises the queue process — allowing clinic staff to register patients,  
assign priorities, estimate waiting times, and track who has been attended to.

## 🖥️ Features

- ✅ **Patient Registration** — Name, age, gender, department, priority, contact
- 🔢 **Auto Queue Number** — Generated by priority (E-001, H-001, N-001, L-001)
- ⏱️ **Wait Time Estimation** — Calculates estimated wait per patient
- 📋 **Live Queue List** — Treeview with colour-coded priorities
- ✔️ **Mark as Seen** — Update patient status in real time
- 📊 **Dashboard** — Summary stats, priority breakdown, department totals
- 🖨️ **Registration Receipt** — Instant on-screen receipt after registration

## 📐 Structured Programming Concepts Used

|Concept               |Where Used                                                               |
|----------------------|-------------------------------------------------------------------------|
|Variables & Constants |`CLR_BG`, `FONT_TITLE`, `DEPARTMENTS`, `queue_counter`                   |
|Data Types            |Strings, integers, booleans, dicts, lists                                |
|Decision Structures   |`validate_patient_input()`, `calculate_wait_time()`                      |
|Iteration (Loops)     |`calculate_wait_time()`, `get_queue_statistics()`, `_refresh_dashboard()`|
|User-Defined Functions|6+ (see below)                                                           |
|GUI Components        |Labels, Entries, Buttons, Comboboxes, Treeview, Radiobuttons             |

### User-Defined Functions (≥ 3 required)

1. `generate_queue_number(priority)` — Creates unique queue codes by priority
1. `calculate_wait_time(priority, position)` — Estimates wait using loops
1. `validate_patient_input(name, age_str, contact)` — Input validation with decision structures
1. `register_patient(...)` — Builds and stores patient record
1. `get_queue_statistics()` — Aggregates queue data using loops
1. `mark_patient_seen(queue_no)` — Updates patient status

## 🔒 Privacy & Ethics

- No data is stored to disk — all records are held in memory during the session only
- No patient data is transmitted externally
- Contact numbers are optional
- System is designed for clinic staff use only, not public-facing
